git init 
     建立一个初始化仓库
git touch 1.txt
     建立一个1.txt文件
echo master >> 1.txt
cat 1.txt
git add -A
git commit -m 's'
    1.txt内容变为master
git branch v1
    建立一个v1branch
git checkout v1
     切换到v1
git echo v1 >> 1.txt
     追加v1进入1.txt
git add 1.txt
git commit -m 'f'
git checkout v1
     提交进master分支
echo 'newmaster' >> 1.txt
git add 1.txt
git commit -m 'd'
      在master上更改并提交1.txt
git merge v1
      此时会出现问题，1.txt的内容会发生冲突，原因是，在merge之前，分别在v1和master上面进行了对1.txt的修改，merge时，会产生冲突。也就是，同时在不同分支修改内容提交后，合并时会产生冲突，必须手动把文件里面的内容改掉。修改内容的方式，是把===,>>>>这些指向或分割的符号删掉，以及对两次修改的内容进行挑选，确定想留下哪些修改然后再进行合并。
